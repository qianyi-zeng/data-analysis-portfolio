# 案例 2：GMV 波动与渠道归因分析

## 背景

行业 GMV 出现周期性波动时，单看总 GMV 很难判断问题来自流量、转化、供给、用户结构、补贴变化还是体验治理。业务需要一套稳定的拆解路径，用于周会复盘和追赶动作制定。

## 分析目标

- 判断 GMV 波动的主因和次因。
- 区分短期节假日影响、平台策略影响、供给变化、补贴变化和体验问题。
- 给出可以落地的运营动作，而不是停留在描述数据涨跌。

## 分析路径

### 1. 先定位结果层变化

| 指标 | 目的 |
| --- | --- |
| 本周日均 GMV | 判断短期业务表现 |
| QTD GMV | 判断季度目标进度 |
| 环比 / 同比 | 区分短期波动和结构性变化 |
| 剩余日均 Gap | 判断追赶压力 |

### 2. 拆渠道

把 GMV 拆成真人直播、数字人直播、短视频、货架及其他。每个渠道继续拆：

- 流量：曝光、看播、播放、点击。
- 转化：CTR、CVR、GPM、OPM。
- 供给：有效开播、开播时长、发文量、动销视频。
- 商品/商家结构：头部供给是否掉量，新供给是否补位。

### 3. 拆用户

新客下滑和老客下滑的含义不同：

- 新客下滑：通常要看补贴、拉新货盘、投放、内容分发。
- 老客下滑：通常要看复购、履约体验、商品品质、价格力。
- 新客短期下滑会在后续周期影响老客基数，需要提前预警。

### 4. 拆费用与资源

补贴或资源投入不能只看 GMV，需要同时看：

- 增量 GMV。
- 补贴消耗。
- ROI。
- 补贴率。
- 预算消耗速度。
- 是否存在低质供给套利。

### 5. 拆体验

GMV 短期增长但 NPS、品退率、差评率恶化，说明增长质量有风险。体验指标用于判断是否要清退低质供给、暂停补贴或调整货盘。

## 结论表达模板

> 本周 GMV 变化主要由 A 渠道贡献，表层原因是流量/转化/供给变化，深层原因是 B 类供给缺失、C 类用户补贴敏感、D 类商品体验波动。建议短期用资源扶持和重点商家排班修复供给，中期补充高复购货盘，长期建立低质供给预警和新商冷启机制。

## 业务价值

- 将“GMV 下滑”拆成可行动的具体问题。
- 避免把所有问题都归因到流量，提升运营动作准确度。
- 能同时服务周报复盘、季度追赶和活动期资源分配。

## 可复用方法

- 指标树：结果指标 -> 渠道指标 -> 过程指标 -> 供给/用户/费用/体验指标。
- 对比框架：本周 vs 上周、本期 vs 去年同期、当前 QTD vs 目标进度。
- 动作闭环：发现问题 -> 定位责任模块 -> 给出动作 -> 下周验证效果。
