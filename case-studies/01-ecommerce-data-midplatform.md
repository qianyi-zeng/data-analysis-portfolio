# 案例 1：电商行业中台数据监控体系

## 背景

某头部内容电商平台的食品家居行业涉及真人直播、数字人直播、短视频、货架等多个交易场景。业务团队需要同时追踪 GMV、订单、流量、转化、供给、补贴、用户新老结构和目标进度，但早期存在口径分散、人工取数耗时、日报周报依赖人工拼接的问题。

## 目标

- 建立一套稳定的行业中台数据监控体系。
- 支持日常经营判断、周会复盘、活动期目标追踪和异常归因。
- 降低重复取数和人工校验成本。

## 我的工作

1. 梳理核心指标口径：支付 GMV、成交 GMV、订单量、曝光、点击、看播、转化率、补贴金额、新老客、有效开播、优质新商等。
2. 按业务决策场景拆分报表：日报看短期波动，周报看趋势和归因，活动报表看目标达成和资源效果。
3. 搭建 Excel 自动化模板：用日期参数、SUMIFS、同比/环比、目标进度和 Gap 计算减少手工更新。
4. 建设 BI 看板：按 GMV、流量、供给、转化、目标进度、商家/主播维度进行拆解。
5. 沉淀数据口径文档和周报取数 SOP，让团队成员可以复用同一套取数逻辑。

## 指标框架

| 层级 | 关键问题 | 指标 |
| --- | --- | --- |
| 结果层 | 业务是否达成目标 | GMV、订单量、QTD 达成率、剩余日均 Gap |
| 渠道层 | 哪个渠道贡献变化 | 真人直播、数字人、短视频、货架 GMV |
| 流量层 | 是流量问题还是转化问题 | 曝光、点击、看播、CTR、CVR |
| 供给层 | 是否有足够供给承接 | 有效开播、发文量、动销视频、优质新商 |
| 用户层 | 增长来自新客还是复购 | 新客数、老客数、复购率、新老客 GMV |
| 费用层 | 增长是否有钱效 | 补贴金额、补贴率、ROI、预算消耗节奏 |
| 体验层 | 增长是否健康 | NPS、品退率、差评率、低质供给占比 |

## 产出

- 3 类渠道日/周报体系。
- 10+ 个业务监控看板。
- 可复用 Excel 取数和汇总模板。
- 周报取数 SOP 与指标口径文档。

## 业务价值

- 日/周报产出时间减少约 5 小时。
- 数据准确率提升至约 98%。
- 管理层和运营同学可以更快定位 GMV 波动原因，并把讨论从“数据对不对”前移到“问题怎么解决”。
